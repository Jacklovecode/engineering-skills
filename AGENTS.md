# Engineering Skills

Use `skills/start/SKILL.md` as the routing entry point.

When a task clearly matches another skill, read that skill before acting:

- `zoom-out`: unfamiliar code area, call graph, data flow, impact range.
- `deepen`: architecture refinement, refactoring opportunities, coupling, testability.
- `brainstorm`: clear goal with multiple candidate approaches, trade-off comparison after relevant context is understood.
- `clarify`: unclear requirements, new features, behavior changes.
- `grill`: stress-test a design, plan, or idea before implementation.
- `plan`: turn confirmed requirements into task layers.
- `execute`: execute a confirmed plan task by task.
- `tdd`: implement behavior changes with behavior-first tests.
- `diagnose`: failing tests, broken builds, unexplained behavior, performance regressions.
- `review`: check requirement fit and engineering quality.
- `finish`: completion gate before claiming completion.
- `git`: version-control operations, commits, pushes, merges, PRs, remotes, cleanup.
- `skill-edit`: create or modify skills.

The skill bodies are written in Chinese. Follow their constraints and gates exactly.
