# 贡献指南

感谢你帮助改进 Engineering Skills。

本项目是面向编码智能体的工程技能包。贡献内容应当提升技能的可靠性、清晰度、跨平台兼容性或验证质量。

## 适合贡献的内容

欢迎提交：

- 新技能，或对现有技能的具体改进。
- 更符合中文语境的技能描述、约束和流程表达。
- 能暴露歧义、不安全行为或流程缺口的验证场景。
- Codex、Claude、Gemini 或其他智能体运行环境的适配改进。
- 帮助用户安装、使用或扩展技能包的文档。

不建议提交：

- 没有具体使用场景的宽泛流程要求。
- 让技能更难执行，但没有提升正确性的改动。
- 本地笔记、私有报告、缓存、备份文件或临时生成文件。
- 在一个 PR 中修改大量无关文件。

## 技能设计要求

新增或修改技能时，请遵守：

1. 技能名使用简短英文，便于跨平台调用。
2. `description` 保持简洁，便于平台索引。
3. 技能主体的约束、检查点和执行流程使用中文。
4. 明确说明这个技能什么时候应该被使用。
5. 明确智能体在声称完成前必须提供什么证据。
6. 优先提供行为示例和验证场景，少写抽象建议。
7. 保持技能目标清晰。如果一个技能承担多个无关任务，应拆分。

## 提交前验证

提交 Pull Request 前，请运行：

```powershell
.\scripts\validate.ps1
.\scripts\simulate-pressure.ps1
```

macOS 或 Linux 可运行：

```bash
./scripts/validate.sh
./scripts/simulate-pressure.sh
```

期望结果：

- 结构校验通过。
- 压力场景验证通过。
- 提交内容不包含 `docs/` 目录中的文件。

## 提交内容规范

不要提交：

- `docs/`
- 本地报告或过程笔记。
- 备份文件。
- 密钥、令牌、凭据或个人本地路径。
- 依赖缓存或临时生成文件。

提交前建议检查：

```bash
git status --short --ignored
git ls-files | grep '^docs/' || true
```

Windows PowerShell：

```powershell
git status --short --ignored
git ls-files | Select-String -Pattern '^docs/'
```

第二条命令不应输出任何已跟踪的 `docs/` 文件。

## Pull Request 要求

每个 PR 应包含：

- 改动摘要。
- 为什么需要这个改动。
- 影响了哪些技能或工作流。
- 已运行的验证命令和结果。
- 仍然存在的风险或后续工作。

请尽量保持 PR 小而清晰。较大的方向性改动，请先创建 Issue 讨论。

## 审查标准

维护者会重点审查：

- 是否符合项目目标。
- 中文表达是否自然、准确。
- 技能触发条件是否清楚。
- 工作流是否完整。
- 是否包含必要的安全检查点和验证证据。
- 是否兼顾多平台使用。

本项目不会合并所有想法。维护目标是让技能包在日常工程使用中保持可靠、清晰、可验证。
