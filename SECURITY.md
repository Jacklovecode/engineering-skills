# 安全策略

## 报告安全问题

如果发现密钥泄露、凭据暴露、供应链风险，或可能伤害用户的漏洞，请不要直接创建公开 Issue 暴露细节。

请优先通过 GitHub 联系仓库维护者。若没有私下联系方式，可以创建公开 Issue，只描述问题类别，不公开敏感内容。

## 范围

安全问题可能包括：

- 误提交的密钥或凭据。
- 不安全的安装或执行说明。
- 可能删除或上传非预期文件的脚本。
- 可能泄露私有材料的 Git 工作流说明。
- 申请了不必要权限的平台配置文件。

## 报告内容

报告时请尽量包含：

- 受影响的文件或工作流。
- 可安全公开的复现步骤。
- 可能造成的风险。
- 已知的修复建议。

维护者应确认有效报告，必要时移除敏感材料，并发布修复或缓解说明。
